## 核心判断

你现在的情况是：兴趣广泛、基础分散、时间有限。
解决方案不是"学更多"，而是**选一个格架，让每个领域互相喂养**。

你的格架：
```
行为设计（为什么人这样选择）
    ↕
经济学（激励机制与决策）
    ↕
AI（技术放大行为干预的手段）
    ↕
CS 基础（理解 AI 底层，不被黑箱锁死）
    ↕
视觉传播（把以上所有东西讲给别人听）
```

每个领域学到的东西，都能用来解释其他领域。这是复利，不是叠加。

## 每日 Routine（最终版）

### 雷打不动（每天，共 35 分钟）

| 时间 | 行动 | 工具 |
|---|---|---|
| 起床后 | 有氧 20-30 min（谭成毅 / 跑步） | 抖音 / 户外 |
| 通勤去 | 听播客 / 英语，到站前 3 min 记 Flomo | 手机 |
| 通勤回 | 同上 | 手机 |
| 睡前 5 min | Daily Note：今天一件事 + 感恩一句话 | Obsidian |

### 每天一个深度学习块（90 分钟，轮换）

| 星期 | 主题 | 具体做什么 |
|---|---|---|
| 一 | 行为设计 / IRP | 推进毕业设计，或读行为科学 |
| 二 | AI 研究 | 研究工具、读摘要、做实验 |
| 三 | 经济学 / 金融 | 读书或看视频 |
| 四 | 计算机基础 | CS50P / Computerphile / 编程练习 |
| 五 | 内容创作 | P 图 + 写 100 字 + 发小红书 |
| 六 | Flomo 整理 | 本周 Flomo 发给 Claude，提炼笔记 |
| 日 | 复盘 | 回顾本周，写下下周一件事 |

## 可迁移性说明

这些技能为什么可迁移：

**创业场景**：用行为设计做产品，用 AI 做工具，用经济学定价和理解市场
**找工作场景**：AI + CS 基础 = 能和工程师对话的设计师；行为设计 = 产品/UX 稀缺背景
**内容创作**：把这些东西讲成别人能懂的故事，这本身就是护城河

你不需要精通每一个。你只需要在每个领域里能提出比大多数人更好的问题。
