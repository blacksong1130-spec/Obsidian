## 速览 · TL;DR

> 作者用 AI 编程（Vibe Coding）做了一个 Chrome/Edge 插件，可以一键抓取 B 站视频字幕并写入 Obsidian 指定目录，弥补官方 Web Clipper 不支持 B 站的缺口，再交给 AI 整理成可读笔记。

## 关键洞见 · Key Insights

- **收藏夹≠知识**：B 站收藏夹打开率低、不能搜索、内容可能消失——本地化才是真正的知识留存。
- **Local REST API 是关键桥梁**：浏览器插件无法直接写文件系统，需要 Obsidian 插件 "Local REST API" 开启本地接口（HTTP 27123）做中转。
- **AI 处理的前提是结构化原料**：原始字幕是噪音，有了字幕文本后才能让 AI 真正总结、翻译、格式化——工具链的价值在于"输入质量"。
- **Git 是 Vibe Coding 的安全网**：AI 编程过程中功能被优化坏后，靠 git 版本回退救场——git 不需要手动操作，AI 会自动提交。
- **多模型协作解 Bug**：Codex 解不了的 Bug，用 Gemini（Opus 4.5 免费额度）分析方案，再把方案喂回 Codex 执行——跨模型分工是实用策略。

## 内容摘要 · Summary

### 为什么需要这个工具

B 站是国内最大的知识学习平台，但 Obsidian 官方 Web Clipper 对 B 站没有任何适配。收藏夹本质上是低效的：打开率低、不可搜索、内容随时可能消失。目标是把看过的视频沉淀到本地知识库，再接入 AI 进行后续整理和搜索。

### 插件核心功能

在任何有字幕的 B 站视频页面点击插件，可以：获取作者上传字幕或 AI 生成字幕（多语言可选）；复制完整 Markdown 内容；下载 SRT 或 TXT 格式；一键写入 Obsidian 指定目录。保存内容包含视频简介、章节目录、嵌入视频链接和字幕全文。

### 安装配置要点

插件已上架 Chrome 和 Edge 插件市场，也可在 GitHub 手动加载未打包版本。关键依赖：Obsidian 插件 **Local REST API**——给 Obsidian 开一个本地 HTTP 接口（推荐 27123 非加密端口，个人电脑本地使用无需加密），浏览器插件通过该接口写入 MD 文件。Windows 版本配置可参考插件 Wiki。

### 结合 AI 的后处理流程

在 Obsidian 中维护一个 `claude.md` 文件，明确规定 AI 的输出格式、命名规则、翻译要求、存储位置。调用 Claude 处理后，原始文件归档至"已归档"文件夹，生成格式化笔记存入内容库——原文不删除，保留原始依据。

### Vibe Coding 经验教训

最难解决的 Bug：字幕抓取时会拿到错误的字幕文件。Codex 反复修改无效，最终换用 Gemini 中的 Opus 4.5 分析出根本原因，将分析方案提供给 Codex 执行才解决。另一个教训：让 AI "优化冗余代码" 反而搞坏了功能，幸亏 AI 在开发过程中自动维护了 git 历史，版本回退救场。
